# Security Policy

We take security seriously in the GhostNode project. As a library designed for synchronization in distributed and enterprise platforms, keeping your data secure and integral is a top priority.

## Supported Versions

Only the latest active release branch receives security updates. We recommend upgrading to the latest version as soon as it is released.

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |
| < 0.1.0 | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in GhostNode, please do **not** open a public issue. Instead, report the vulnerability privately to help protect users of the library.

Please report vulnerabilities by sending an email to the maintainer: **mathantkumar@gmail.com** (or the email listed on the project's profile).

In your report, please include:
- A description of the vulnerability and its potential impact.
- Detailed step-by-step instructions (and ideally a minimal proof-of-concept project or test case) to reproduce the behavior.
- Any suggestions for remediation or patches.

## Our Security Response Process

Upon receiving a private vulnerability report, the maintainers will:
1. Acknowledge receipt of the report within 48 hours.
2. Investigate the issue to confirm the vulnerability and assess its severity.
3. Keep the reporter updated as a fix is developed.
4. Prepare a security patch release.
5. Publish a security advisory detailing the vulnerability and the fix once the release is available.
